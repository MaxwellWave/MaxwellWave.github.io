    为什么 always 行为级建模时敏感信号使用posedge clk

    前言

    在写verilog代码的时候，笔者突然想到为什么不管在书上还是例程上在使用always语句块进行行为级建模的时候敏感信号都用的是always@（posedge clk or negedge rst_n），为什么采用时钟上升沿以及复位信号下降沿。复位信号为什么选用下降沿笔者之后再更新，咱们先来聊聊为什么选用时钟上升沿？
    先说结论使用时钟上升沿的目的是便于综合工具知道你写的是一个D触发器。从Xilinx7系列FPGA的设计文档中可以找到D触发器的类型有四种分别是FDCE（异步复位）、FDPE（异步置位）、FDRE（同步复位）、FDSE（同步置位）

    xilinx内部触发器说明

    FDCE( D Flip-Flop with Clock Enable and Asynchronous Clear)

    该触发器时钟使能异步复位，当复位信号为高时，输出为低且不受其他端口信号影响；当时钟使能信号CE为高且异步复位信号为低时，才会在时钟上升沿的时候将输入端口D的数据传递给输出端口Q；当时钟使能信号CE为低时，触发器用作锁存器，不受时钟信号干扰，输出保持不变。

    FDPE(D Flip-Flop with Clock Enable and Asynchronous Preset)

    该触发器时钟使能异步置位，当置位信号为高时，输出为高且不受其他端口信号影响；当时钟使能信号CE为高且异步置位信号为低时，才会在时钟上升沿的时候将输入端口D的数据传递给输出端口Q；当时钟使能信号CE为低时，触发器用作锁存器，不受时钟信号干扰，输出保持不变。

    FDRE(D Flip-Flop with Clock Enable and Synchronous Reset)


    该触发器时钟使能同步复位，当在时钟上升沿采集到复位信号R为高时，输出为低且不受其他端口信号影响；当时钟使能信号CE为高且同步复位信号为低时，才会在时钟上升沿的时候将输入端口D的数据传递给输出端口Q；当时钟使能信号CE为低时，触发器用作锁存器，不受时钟信号干扰，输出保持不变。

    FDSE( D Flip-Flop with Clock Enable and Synchronous Set)

    该触发器时钟使能同步置位，当在时钟上升沿采集到复位信号R为高时，输出为高且不受其他端口信号影响；当时钟使能信号CE为高且同步复位信号为低时，才会在时钟上升沿的时候将输入端口D的数据传递给输出端口Q；当时钟使能信号CE为低时，触发器用作锁存器，不受时钟信号干扰，输出保持不变。

    代码实现不同类型触发器

    从上面Xilinx关于触发器的介绍中我们可以知道，不论是那哪种类型的触发器，其数据的传输都发生在时钟的上升沿，这也就是为何always里面使用posedge clk。笔者写了一段如下代码可以综合出不同类型的触发器

    module assginment1
(
    input			clk,
    input			rstn,
    input           in,
    output	reg		out1,out2,out3,out4
);

always@(posedge clk or negedge rstn)	begin	//FDCE
    if(!rstn)
        out1	<=	1'b0;
    else
        out1	<=	in;
end

always@(posedge clk or negedge rstn)	begin	//FDPE
    if(!rstn)
        out2	<=	1'b1;
    else
        out2	<=	in;
end

always@(posedge clk)	begin                   //FDRE
    if(!rstn)
        out3	<=	1'b0;
    else
        out3	<=	in;
end
    
always@(posedge clk)	begin                   //FDSE
    if(!rstn)
        out4	<=	1'b1;
    else
        out4	<=	in;
end
endmodule

    总结

    综上所述在always里面描述时序电路时使用posedge clk是便于综合工具的识别触发器，当然也可以使用negedge，但是这会平白浪费一个反相器。
