    Windows下如何搭建自己的博客

    1 科学上网

    1.1 stream加速器

    笔者在跟着B站程序羊搭建博客的过程中遇到的最大的问题就是CMD里面无法访问GitHub，即便用了Edge浏览器的梯子可以在浏览器中访问GitHub但也无法在CMD使用hexo将博客部署在GitHub的仓库上，会出现无法访问到GitHub的错误提示。

    笔者也尝试过修改DNS添加GitHub的站点也未能解决该问题，最后是使用了 stream++ 对GitHub加速后才成功，但是笔者发现 stream++ 其实也是通过修改host(其路径为C:\Windows\System32\drivers\etc)解决

    注意如果是手动修改host后需要在CMD内通过以下代码刷新DNS

        ipconfig /flushdns

    1.2 挂在梯子终端访问

    搭载梯子后虽然浏览器可以访问GitHub，但是终端还是不行，此是在windows下打开cmd输入以下命令，其中最后五个数字是自己梯子的端口号

    set http_proxy=http://127.0.0.1:33210
set https_proxy=http://127.0.0.1:33210

    设置完成之后可通过以下命名查看是否能够访问成功，如果可以得到一大堆数据反馈则说明成功，如果长时间没反应说明失败

    curl www.google.com

    需要注意此方法一旦关闭终端就会失效，需要每次打开终端进行设置

    2 安装过程

    如果能保证命令行访问到GitHub，建立博客的过程就十分简单，大家可以参考码游菌和薯叶的两篇文章介绍，笔者在此就不再赘述，仅介绍一下笔者在安装过程中所使用到的指令以及遇到的一些问题而两篇文章中没有介绍到的

    2.1 软件下载

    2.1.1 下载并安装node.js

    地址：https://nodejs.org/en/

    2.1.2 下载并安装git

    地址：https://git-scm.com/download/win

    2.1.3 命令行安装cnpm

    npm install -g cnpm --registry==https://registry.npm.taobao.org

    2.1.4 命令行安装hexo

    cnpm install -g hexo-cli

    2.1.5 安装版本验证及配置

    #以下命令为windows下cmd, linux系统下需先获取sudo权限
    node -v                                 #查看nodejs版本
    npm  -v                                 #查看npm版本
    git  -v                                 #查看git版本
    cnpm -v                                 #查看cnpm版本
    hexo -v                                 #查看hexo版本

    npm config get registry                 #查看是否替代源

    git config --global user.email "xxx"    #设置GitHub邮箱地址
    git config --global user.name  "xxx"    #设置GitHub用户名
    git config user.email                   #查看git邮箱
    git config user.name                    #查看git用户名

    cnpm install hexo-deployer-git --save       #在blog目录下安装git部署插件
    cnpm install hexo-renderer-marked --save    #hexo远端部署图片插件

    2.2 博客创建过程

        mkdir blog              #新建博客目录
    cd blog                 #进入博客文件夹
    hexo init               #生成初始化博客
    hexo new "第一篇博客"   #新建第一篇博客
    hexo clean              #清理清除缓存文件 (db.json) 和已生成的静态文件 
    hexo generate           #生成博客

##  注意：对_config.yml进行配置后才可进行远端部署

    hexo deploy             #远端部署

    git clone 主题链接       #克隆主题

    2.3 SSH免密远端部署

    使用薯叶远端部署的方法，笔者在输入登入密码的时候一直显示错误，故在远端部署之前可以现根据码游菌的方法配置SSH免密登入，需要注意的是Windows下无法下述命令测试SSH配置成功

        $ ssh -T git@github.com

    直接使用 hexo d 命令进行远端部署，如果无需输入密码就部署成功就说明SSH免密配置成功

    注意： 远端部署前需要先对git设置访问的邮箱和用户名

    2.3.1 操作步骤

    第一步：进入C:\User\x\路径，查看是否存在.ssh文件，如果存在则删除

    第二步：在C:\User\x\路径下，打开终端输以下命令，输入后连敲三次回车，出现The key's randomart image is 说明ssh密匙生成成功

    ssh-keygen -t rsa -C <github邮箱地址>

    第三步：打开新生成的C:\User\x\.ssh文件夹，使用记事本打开id_rsa.pub文件，并复制里面的内容

    第四步：打开个人的GitHub主页，进入Setting -> SSH and GPG keys -> New SSH key ，将复制的内容粘贴进去，保存即可。此时公钥就添加好了，在博客目录下进行远端部署即可检查是否成功

    3 VScode 编辑markdown文本

    相较于使用Typora编辑markdown文本，笔者更喜欢能看到代码且同时能预览结果的VScode,当然要想VScode实现相应功能则需要进行一些配置

    3.1 实时预览

    预览功能只需要下载插件Markdown Preview Github Styling即可，无需多余的配置，之后便可以通过快捷键和右上角的图标进行实时预览，效果如下

    3.2 图片部署

    在md文件中插入图片的语法为![]()其中方括号是图片描述，圆括号是图片路径。然而在远端部署的时候会出现图片无法加载的情况，故需要对hexo以及vscode都进行一些相应的配置。

    hexo

    首先需要使用命令cnpm install hexo-renderer-marked --save下载 hexo-renderer-marked 插件，然后对hexo的_config.yaml配置文件进行如下配置

        post_asset_folder: true #使得source文件夹内生成一个与md文件同名的文件夹用于存放该文件的图片
    marked:
        prependRoot: true
        postAsset: true

    之后只需要将图片放进与博客名同名的文件夹中再使用![](img.jpg)即可将图片部署到远端上

    VScode

    VScode本身不具备粘贴图像的功能，故需要先下载Markdown Paste插件使得其具有粘贴图像的功能,然而该插件默认将图片保存在当前文件夹目录下，故我们需要将其默认路径改为当前博客名的文件夹。

    目前此插件设定的路径有如下预定义变量

    $ {workspaceRoot}-在VS Code中打开的文件夹的路径
$ {fileBasename}-当前打开的文件的基本名称
$ {fileBasenameNoExtension}-当前打开的文件的基本名称，没有文件扩展名
$ {fileExtname}-当前打开的文件的扩展名
$ {fileDirname}-当前打开的文件的目录名

    故需要将其路径变量修改为fileBasenameNoExtension

    4 增加看板娘功能

    4.1 准备工作

    打开博客主目录下的package.json文件，该文件中记录了hexo使用的一些插件和支持，检查该文件中是否有"hexo-helper-live2d": "^3.1.1"。如果有的话可以使用以下命令卸载，防止之前安装的看板娘造成影响

    npm uninstall hexo-helper-live2d

    然后再使用此命令安装依赖

    npm install --save hexo-helper-live2d

    4.2 模型文件下载

    模型文件可以在此处下载，然后在此处进行预览，之后挑选想要的模型并将模型文件加放进博客主目录中的node_modules文件夹中。

    注意：有时候看板娘无法正常显示，很有可能是因为模型文件被删除了，此时重新把模型文件复制进node_modules文件夹中即可

    4.3 _config.yml配置

    将如下代码添加进博客主目录中的_config.yml文件或者主题的_config.yml文件中

    ## Live2D看板娘
live2d:
  # 是否要开启看板娘，false 就是关闭
  enable: true
  pluginModelPath: assets/
  model:
    # 这里填入你想要显示的模板的名称，也就是你加入到 node_modules 文件中内个模型的名字
    use: live2d-widget-model-hijiki
  display:
      # 这里可以调整模型显示的位置，可以写 left 或者 right，将模型显示在左下角或者右下角
    position: left
    # width 和 height 属性可以调整整个模型显示的大小，这里我的小猫在 1:2 的比例下都能显示出来
    width: 150
    height: 300
  mobile:
    # 是否要在手机端显示，true 就是要，false 就是不要
    show: false   
  rect:
    opacity:0.7
