    Windows下如何搭建自己的博客

    1 科学上网

    笔者在跟着B站程序羊搭建博客的过程中遇到的最大的问题就是CMD里面无法访问GitHub，即便用了Edge浏览器的梯子可以在浏览器中访问GitHub但也无法在CMD使用hexo将博客部署在GitHub的仓库上，会出现无法访问到GitHub的错误提示。

    笔者也尝试过修改DNS添加GitHub的站点也未能解决该问题，最后是使用了 stream++ 对GitHub加速后才成功，但是笔者发现 stream++ 其实也是通过修改host(其路径为C:\Windows\System32\drivers\etc),下图为未修改前host的文件内容

    下图为开启 stream++ 之后host的文件内容

    注意如果是手动修改host后需要在CMD内通过以下代码刷新DNS

        ipconfig /flushdns

    2 安装过程

    如果能保证命令行访问到GitHub，建立博客的过程就十分简单，大家可以参考码游菌和薯叶的两篇文章介绍，笔者在此就不再赘述，仅介绍一下笔者在安装过程中所使用到的指令以及遇到的一些问题而两篇文章中没有介绍到的

    #以下命令为windows下cmd, linux系统下需先获取sudo权限
    node -v                                 #查看nodejs版本
    npm  -v                                 #查看npm版本
    git  -v                                 #查看git版本
    git config --global user.email "xxx"    #设置GitHub邮箱地址
    git config --global user.name  "xxx"    #设置GitHub用户名
    git config user.email                   #查看git邮箱
    git config user.name                    #查看git用户名


    npm install -g cnpm --registry==https://registry.npm.taobao.#安装cnpm管理器并替换源
    npm config get registry                                     #查看是否替换源
    cnpm -v                                                     #查看cnpm版本
    cnpm install -g hexo-cli                                    #安装hexo架构
    hexo -v                                                     #查看hexo版本

    mkdir blog              #新建博客目录
    cd blog                 #进入博客文件夹
    hexo init               #生成初始化博客
    hexo new "第一篇博客"    #新建第一篇博客
    hexo clean              #清理清除缓存文件 (db.json) 和已生成的静态文件 
    hexo generate           #生成博客

    cnpm install hexo-deployer-git --save #在blog目录下安装git部署插件

#----------------------注意：对_config.yml进行配置后才可进行远端部署-----------------------#

    hexo deploy             #远端部署

    git clone 主题链接       #克隆主题
    cnpm install hexo-renderer-marked --save #hexo远端部署图片插件

    2.1 查看cnpm的镜像源

    在使用命令

        npm install -g cnpm --registry==https://registry.npm.taobao.org

    安装cnpm并切换淘宝源的时候，可以使用以下命令查看是否切换成功

        npm config get registry # 查看现在的源

    2.2 SSH免密远端部署

    使用薯叶远端部署的方法，笔者在输入登入密码的时候一直显示错误，故在远端部署之前可以现根据码游菌的方法配置SSH免密登入,需要注意的是Windows下无法下述命令测试SSH配置成功

        $ ssh -T git@github.com

    直接使用 hexo d 命令进行远端部署，如果无需输入密码就部署成功就说明SSH免密配置成功

    注意： 远端部署前需要先对git设置访问的邮箱和用户名

    3VScode 编辑markdown文本

    相较于使用Typora编辑markdown文本，笔者更喜欢能看到代码且同时能预览结果的VScode,当然要想VScode实现相应功能则需要进行一些配置

    3.1实时预览

    预览功能只需要下载插件Markdown Preview Enhanced即可，无需多余的配置，之后便可以通过快捷键和右上角的图标进行实时预览，效果如下

    3.2图片部署

    在md文件中插入图片的语法为![]()其中方括号是图片描述，圆括号是图片路径。然而在远端部署的时候会出现图片无法加载的情况，故需要对hexo以及vscode都进行一些相应的配置。

    hexo

    首先需要使用命令cnpm install hexo-renderer-marked --save下载 hexo-renderer-marked 插件，然后对hexo的_config.yaml配置文件进行如下配置

        post_asset_folder: true #使得source文件夹内生成一个与md文件同名的文件夹用于存放该文件的图片
    marked:
        prependRoot: true
        postAsset: true

    之后只需要将图片放进与博客名同名的文件夹中再使用![](img.jpg)即可将图片部署到远端上

    VScode

    VScode本身不具备粘贴图像的功能，故需要先下载Markdown Paste插件使得其具有粘贴图像的功能,然而该插件默认将图片保存在当前文件夹目录下，故我们需要将其默认路径改为当前博客名的文件夹。

    目前此插件设定的路径有如下预定义变量

    $ {workspaceRoot}-在VS Code中打开的文件夹的路径
$ {fileBasename}-当前打开的文件的基本名称
$ {fileBasenameNoExtension}-当前打开的文件的基本名称，没有文件扩展名
$ {fileExtname}-当前打开的文件的扩展名
$ {fileDirname}-当前打开的文件的目录名

    故需要将其路径变量修改为fileBasenameNoExtension
