    Linux命令学习-find

    0 描述

    find命令递归遍历所指定的目录树，针对每个文件依次执行find命令中的表达式，表达式首先根据逻辑运算符进行结合，然后依次从左至右对表达式求值。使用该命令时，如果不设置任何参数，则该命令将在当前目录下查找子目录与文件，并将查找到的子目录和文件全部显示。该命令支持按文件、文件夹、名称、创建日期、修改日期、所有者和权限进行搜索；支持使用-exec对找到的文件和文件夹执行其他的UNIX命令

    1 语法

    #语法
$ find [where to start searching from] [expression determines what to find] [-options] [what to find]

#实例
$ find . -type f -name "*.v" -exec ls -l {} \;
#找到名称为包含.v的一般文件,打印后并显示出详细信息

    find命令根据- () !判断path和expression，之前为路径，之后为操作表达式。如果路径是空字符串则默认使用当前路径，如果表达式为空则默认使用预设表达式-print。即当在终端输入find命令后会打印出当前目录下的子目录及文件。

    find命令的基本的表达式分为三类，分别是设置项(option)、测试项(test)、动作项(action)，这三类又可以通过逻辑运算符(operator)组合在一起形成更大更复杂的表达式。

    • 设置项（如-depth,-maxdepth等）针对这次查找任务，而不是仅仅针对某一个文件，设置项总是返回true；
    • 测试项(test)则不同，它针对具体的一个文件进行匹配测试，如-name,-num,-user等，返回true或者false；
    • 动作项(action)则是对某一个文件进行某种动作（最常见的如-print），返回true或者false。

    find命令预置-print参数，使得找到的文件为标准输出，故可以结合管道符号|与其他命令结合使用

    1.1 逻辑运算符

    find 命令支持以下逻辑运算符

    • -and，简写-a，逻辑与
      当使用多个表达式时，通过-a连接，表示必须满足多个表达式的条件，find才会输出结果。
      注意：当存在多个表达式而不输入逻辑运算符时，默认使用-and
    #格式
$ find path expr1 expr2 ...
$ find path expr1 -a expr2 -a ...
$ find path expr1 -and expr2 -and ...

实例
$ find ./ -type f -name "*.v"       #查找当前目录下文件名含.v的所有普通文件
$ find ./ -type f -a -name "*.v"    #同上
$ find ./ -type f -and -name "*.v"  #同上
    • -or，简写-o，逻辑或
      当使用多个表达式时，通过-o连接，表示只需要满足多个表达式中的一个条件，find就会输出结果
    #格式
$ find path expr1 -o expr2
$ find path expr1 -or expr2

#实例
$ find . -name "*.c" -o -name "*.h"    #找出当前目录下文件名包含`.c`与`.h`的文件n/
$ find . -name "*.c" -or -name "*.h"   #同上
    • -not，简写!，逻辑非
      当在表达式前面添加!或者-not表示find输出与当前表达式相反的结果
    #格式
$ find path ! expr1
$ find path -not expr1

#实例
$ find ./ -not -name "*.sv" #查找当前目录下文件名不含.sv的文件
    • 逗号运算符,
      表达式使用,隔开时，输出时会忽略expr1的值，只输出expr2的值。man手册中提到会与-fprintf
    #格式
$ find path expr1 , expr2
#实例
$ find ./ -name "*.sv" , -name "*.v" #输出当前目录下文件名包含.v的文件

    1.2 逻辑运算符的优先级

    以下逻辑表达式的优先级依次递减

    1. ( expr )
      注意使用此括号提高表达式的优先级时必须加上转义符\，即\( ... \)
    2. ! expr
    3. not expr
    4. expr1 expr2
    5. expr1 -a expr2
    6. expr1 -and expr2
    7. expr1 -o expr2
    8. expr1 -or expr2
    9. expr1 , expr2

    注意
    前文也提到多个表达式之间如果不特别声明逻辑运算符，则具有隐式的逻辑与，其优先级比逻辑或高，故在使用过程中，对于使用了逻辑或和逻辑与（其余比-o优先级高的表达式）时，切记要使用（ ）提高表达式的优先级。如下例

    
$ find . -name afile -o -name bfile -print          #只输出bfile文件
$ find . \( -name afile -o -name bfile \) -print    #输出afile和bfile文件

    2 参数说明及实例

    find命令可使用的表达式有二三十个，此处仅介绍常用的部分

    2.1 文件名匹配参数-name和-iname

    -name用于查找指定字符串作为寻找文件或目录的范本样式；-iname的功能与-name一致，只是 -name区分大小写而-iname不区分大小写。

    需要注意find命令查找的匹配项仅支持UNIX Shell风格中的fnmatch函数通配符，它们并不等同于正则表达式。Shell风格通配符所使用的特殊字符如下

    通配符 功能描述
    * 匹配任意的字符串
    ？ 匹配任意的单个字符
    [ ] 匹配中括号中的任意单个字符
    [! ] 匹配任意不在中括号中的字符 
    $ find . -name "file.v"         #查找当前目录下文件名为file.v的文件
$ find . -name "*.v"            #查找当前目录下所有文件名为中含.v的文件
$ find . -name "?.v"            #查找当前目录下文件名为单个字符命名的.v文件
$ find . -name "*.[ch]"         #查找当前目录下所有以.c和.h命名的文件
$ find . -name "*.[!ch]"        #查找当前目录下所有不以以.c和.h命名的文件

    2.2 文件类型匹配参数 -type

    -type参数用于根据文件的类型进行查找，比如有时候需要对文件夹和文件进行区分查找就需要使用到此函数，常用的文件类型有以下几种

    #格式
$ find . -type f  #查找当前路径下的一般文件  

d : 目录型文件
f : 普通文件
l : 链接文件
c : 字符文件

    2.3 支持正则表达式参数-regex pattern

    find可以使用-regex参数使得匹配项pattern支持正则表达式。需要注意的时-regex不是匹配文件名，而是匹配完整的文件名（包括路径）。例如，当前目录下有一个文件./fubar3，如果使用用”f.*r3”来匹配，将查找不到任何结果，正确的方法是使用.*bar.或者.*b.*3来匹配。

    -iregex表达式功能与-regex一致，只是其区分大小写

    PS : 上文中提高使用-regex参数后，pattern就支持正则表达式，故其中的元字符都为正则表达式的含义。即 .表示任意的单个字符，*为匹配前面的子表达式零次或多次，则.*即可表达当前文件中的路径

    #实例
$ find . -regex ".*\(\.v$\|.sv$\)"  #查找当前目录下.v和.sv的文件
#实际上当习惯使用管道符合后，可通过find结合grep命令使用正则表达式查找
$ find . | grep "\(\.v$\|\.sv$\)"   #查找当前目录下.v和.sv的文件

    2.4 调用外部命令参数-exec和-ok

    当不使用管道符合|调用外部命令时，find也可以使用-exec和-ok 参数使其支持调用外部参数。

    -exec [command] {} \;

    该参数是把 find 命令的结果交给由-exec调用的[command]来处理。注意这里的{}和\;是标准格式，只要执行”-exec”选项，这两个符号必须完整输入。 其中{}就代表 find 命令的査找结果，\为转义符号，;为停止符号

    -ok用法与-exec命令一致，只是-ok每执行一次命令都会发出一次是否确认的请求

    $ find ./test -name "*.v" -exec ls -l {} \; #找出test目录下所有.v的文件，并列出详细信息
$ find $HOME/. -name "*.txt" -ok rm {} \;   #找出自己家目录下所有的.txt文件并删除
$ find . -type f -mtime +30 -name "*.log" -exec cp {} old \;#将30天前的.log文件移动到old目录中

    3 总结

    find命令有很多种用法，很难一一讲完，此文主要是记录在使用到的一些参数中所有到的一些问题，经过查找解决后记录下来了。博主在使用find命令时发现很多参数其实可以通过其他方式解决，比如说-regex参数可以通过管道符合结合grep函数实现，同理-exec也是如此。

    find命令还支持根据文件的时间、大小、所有权等进行查找，本文暂未介，待以后若使用了再行记录
