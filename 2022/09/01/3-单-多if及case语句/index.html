    条件分支语句说明

    0 前言

    笔者接触IC&FPGA相关知识将近一年了，在写Verilog代码的时候始终无法体会到知乎上众多大佬所说的心中有电路的境界，近来在学习中国大学MOOC课程《芯动力——硬件加速设计课程》中关于条件分支语句部分。通过实验测试后，感觉能够稍微体会到写出来的代码会综合成何种电路，故写下此文用已记录。

    Verilog的条件分支语句有三种： 单if语句、多if语句、case语句

    1 单if语句

    1.1 单if语句的优先级及电路

    单if语句通常会被逻辑综合成多个二选一的选择器组合而成，其具有隐含的优先级，优先级由上往下，只能在initial和always语句中使用。

    故对于if-else语句结构都是通过二选一的数据选择器搭建而来

    always@(a or b or c or d or sel0 or sel1 or sel2 or sel3)
    begin
        if(sel3)
            z	=	d;
        else if(sel2)
            z	=	c;
        else if(sel1)
            z	=	b;
        else if(sel0)
            z	=	a;
        else
            z	=	0;
    end

    其中d具由最高优先级依次递减，a的优先级最低。

    综合出来的电路与我们心中的电路一致。

    1.2 if-else语句的优先级对电路的影响

    以一个简单的计数器电路为例

    //正确示例
always@(posedge clk or negedge rst_n)
    begin
        if(!rst_n)
            cnt <=  4'd0;
        else if(cnt == 4'd20 && flag == 1'b1)
            cnt <=  4'd0;
        else if(flag == 1'b1)
            cnt <=  cnt + 1;
        else
            cnt <=  cnt;
    end

//错误示例
always@(posedge clk or negedge rst_n)
    begin
        if(!rst_n)
            cnt <=  4'd0;
        else if(flag == 1'b1)
            cnt <=  cnt + 1;
        else if(cnt == 4'd20 && flag == 1'b1)
            cnt <=  4'd0;
        else
            cnt <=  cnt;
    end

    上例代码中，正确示例cnt在flag为高时开始计数，当计数到20时会归零后重新计数；而错误示例由于cnt+1的优先级高于清零的优先级，所以错误示例中cnt会一直加1且不会清零

    1.3 if-else语句对电路的优化

    在设计中需要需求进行分析，采用if-else语句映射的硬件结构，合理设计。

    此时需要你设计一个电路，当flag=1时value=A+B,否则value=C+D。我们应该如何进行设计？

    笔者的第一反应是使用一个二选一的选择器进行实现

    if(flag)
    value   =   A + B;
else
    value   =   C + D;

    有了之前的基础，我们可以很容易的画出此段代码描绘的电路

    但是由于加法器会比选择器复杂的多，所需要的面积也更大，故可使用如下代码重构if-else映射的硬件结构

    if(flag)
    begin
        op1 = A;
        op2 = B;
    end
else
    begin
        op1 = C;
        op2 = D;
    end

assign  value = op1 + op2;

    此段代码描绘的电路如下图所示

    该方法减少了一个加法器，使得电路的面积得以减少，但是控制通道信号flag的延迟却从一个加法器变成了一个加法器和数据选择器之后，如果flag信号到来的比较晚，则此电路的性能就会比前一种电路的性能更差。
    故需要根据设计的目标小心设计：选择先“加”后“选”还是先“选”后“加”。

    2 多if语句

    2.1 多if语句的优先级及电路

    多if语句与单if语句类似，具有优先级，且消耗二选一组合逻辑器件。与单if语句不同的是其最高优先级给到最迟到达的信号。以下述代码为例

    always@(a or b or c or d or sel0 or sel1 or sel2 or sel3)
    begin
        z = 0;  //当所有条件不符时z=0；不加则会生成latch
        if(sel0)
            z	=	a;
        if(sel1)
            z	=	b;
        if(sel2)
            z	=	c;
        if(sel3)
            z	=	d;
    end

    alwsys语句中描述组合逻辑电路使用阻塞赋值，其内部语句顺序执行，故其最高优先级是sel3决定的d，最低优先级是sel0决定的a。故而可以画出此段代码描述的电路，因调整了优先级顺序，故于上文中单if语句的电路一致。

    与vivado综合电路一致

    2.2 当多if语句在时序电路中时的电路

    我们将上述代码修改为如下代码描述的时序逻辑电路

    always@(posedge clk)
    begin
        z <= 0;
        if(sel0)
            z	<=	a;
        if(sel1)
            z	<=	b;
        if(sel2)
            z	<=	c;
        if(sel3)
            z	<=	d;
    end

    首先我们要知道在时序逻辑电路中使用非阻塞赋值，并且在always语句块结束时将右端表达式的值赋值给左端。我们可以推断出上述代码描述时序电路的优先级与前面的组合逻辑电路一致，不同的地方在于输出端口加了一个触发器，故而我们可以画出其电路

    与vivado综合电路一致

    2.3 多if语句的应用场景

    若某些设计中，有些信号要求先到达(如关键便能信号、选择信号等)，而有些信号需要后到达(如慢速信号、有效时间较长的信号等)，此时则需要使用if..if...结构。然而其实此类场合也可以选择使用if-else,故在设计过程中尽可能避免使用if..if...结构。

    3 case语句

    3.1 case语句的优先级及电路

    case语句当条件互斥时是没有优先级，但是条件选项不要求互斥。虽然这些条件选项是并发比较的，但执行效果是谁在前且条件为真谁被执行

    case语句描述的电路是一个多路选择器。

    always@(a or b or c or d or sel0 or sel1)
    begin
        case({sel0,sel1})
            2'b00: z = d;
            2'b01: z = c;
            2'b10: z = b;
            2'b11: z = a;
            default: z =0;
        endcase
    end

    以上述代码为例其描述的电路是一个条件选项互斥的四选一数据选择器，电路如下所示

    与vivado综合电路一致

    当将条件选项修改成非互斥时，我们可以看看综合出来的电路是怎样的

    always@(a or b or c or d or sel0 or sel1)
    begin
        case({sel0,sel1})
            2'b00: z = d;
            2'b01: z = c;
            2'b01: z = b;
            2'b11: z = a;
            default: z =0;
        endcase
    end

    如上述代码，笔者将第三个条件选项修改成与第二个条件选项一致，通过vivado综合出来的电路如下图所示

    可以看到此时b是永远无法赋值到z的，同时由于{sel0,sel1}可表示4种情况，而case中的条件选项未完备，故而会出现一个default接口，将z置零。

    3.2 case语句的应用场景

    相对于if-else语句只有两个分支而言，case语句是一种可实现多路分支选择控制的语句，比if-else语句更为直观和方便。一般的，case语句多用于多见译码电路的设计，如描述译码器、数据选择器、状态机及微处理器的指令译码等。

    3.3 casez与casex的用法

    用前提醒： casez与casex都不可综合，多用于仿真。

    casex、 casez 语句是 case 语句的变形，用来表示条件选项中的无关项。

    casex 用 “x” 来表示无关值，casez 用问号 “?” 来表示无关值。

    两者的实现的功能是完全一致的，语法与 case 语句也完全一致。

    3.4 full-case和parallel-Case 综合器指令

    在理论上使用case语句对于不符合条件选项的我们可以使用default，但是实际应用中可能存在没有default的情况存在。

    以交通控制红绿灯为例，只有红、绿、黄三种情况，不会出现第四种，故使用2bit就可表示所有情况，但会多出一种情况，综合的时候就会产生latch。故可使用full-case告诉综合器，当前case结构所列条件已完备，从而消除latch。

    PS:其实也可以使用default,既然实际上不可能出现满足default条件的情况，那么使用了也不会导致系统输出失常，也保证不会出现latch

    //会出现latch的情况
always@(*)
    case(sel) 
        2'b00:  y = a;  //红
        2'b01:  y = b;  //绿
        2'b10:  y = c;  //蓝
    endcase
endmodule

//不出现latch的情况
always@(*)
    case(sel) //synopsys full_case
        2'b00:  y = a;
        2'b01:  y = b;
        2'b10:  y = c;
    endcase
endmodule

//不出现latch的情况
always@(*)
    case(sel) 
        2'b00:  y = a;  //红
        2'b01:  y = b;  //绿
        2'b10:  y = c;  //蓝
        default: y = 0; //实际中不会出现此条件
    endcase
endmodule
