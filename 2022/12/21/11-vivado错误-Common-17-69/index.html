    [Common 17-69] Common failed

    0 问题描述

    在使用vivado对工程进行布局布线时可以通过，但是在Write Bitstream的过程中却失败，并提示如下错误

    1 错误原因

    根据错误的提示可以看到之所有无法生成bit文件是由于bd6e_mac_0_core不允许被生成bit文件，其原因是以太网模块中的一个 IP 没有 license 权限。点击vivado界面上方的Reports>-Report IP Status查看IP的license状态，如下图所示

    如图所示可以看到axi_ethernet下例化了一个bd_bd6e_mac_0，其IP名为Tri Mode Ethernet MAC，状态为Design_Linking，说明此IP只能用来综合以及布局布线，不能够用来生成bit文件

    XLINX IP 状态类型说明

    include : 完全免费

    design linking : 允许用户进行综合、布局布线、仿真，但不允许生成bit文件

    Hardware Evaluation : 允许用户生成bit文件，但是会有时间限制，一定时间后就会关闭IP的功能

    Key for Production Use ：购买后生成的license，全部功能都可以实现

    2 解决方法

    对于此类情况就去要去官网申请IP的license，申请地址点击此处，进入后需先登入xilinx账号，登入后界面如下

    第一步：点击Search Now，输入IP的名称

    第二步：在Certificate Based License下找到刚刚添加的IP

    第三步：点击Generate Node-Locked License出现如下界面

    第四步：点击ADD a host添加一个主机，如果之前此账号已经添加过主机可直接选用。点击后会弹出如下界面

    第五步：在Host Name输入此设备名称，可自定义；在Operating System选择设备的操作系统；在Host ID Type处选择Ethernet MAC;在Host ID Value处输入设备连接的物理地址，可在终端输入ipconfig/all查询，选择其中一个即可，亦可打开vivado进入license管理界面，点击view Hosr Informatation查询，选择其中一个ID输入即可。

    第六步：输入完整信息后点击Add

    第七步：之后依次弹出界面，点击Next和Accept即可，直到弹出以下窗口，说明IP已经申请完毕并发送至邮箱，可到邮箱处进行下载

    此外，可点击界面左下角处进行下载

    下载后的许可文件如下

    将许可文件通过manage license复制进vivado后即完成所有步骤。

    3 IP 状态验证

    进入manage license >- View license status可以查询到IP的状态

    同时重启工程后再次打开Report IP Status可以查询到IP的状态已经变为Hardware_Evaluation

    之后再对工程重新综合布线即可生成bit文件
