    1 开机启动加速

    无论是物理机还是虚拟机，当安装好后我们会发现其启动速度较慢。此时可以使用system-analyze命令去查找开机启动项。首先可以通过man命令查询system-analyze的使用方法。比较常用到的参数就是blame,通过man手册中查询可知，system-analyze blame用来显示开机启动项的时间，从而优化开机启动的时间

    $ systemd-analyze blame
1.963s NetworkManager-wait-online.service
1.094s snapd.service
 937ms dev-sda3.device
 606ms snapd.seeded.service
...

    可以看到输入完命令后终端会显示开机启动项的时间，其中最长时间的应该是plymouth-quit-wait.service,博主已经将其关闭。关闭与开启服务命令如下

    $ sudo systemctl disable <需要关闭的服务>
$ sudo systemctl enable  <需要开启的服务>
$ #实例
$ sudo systemctl disable NetworkManager-wait-online.service
$ sudo systemctl enable NetworkManager-wait-online.service

    但是当使用命令

    sudo systemctl disable plymouth-quit-wait.service

    禁用plymouth-quit-wait.service时，却发现disable并不能彻底将其禁用，只能使用比disable更强力的mask来禁用。两者的区别可通过man systemctl查询。

    $ sudo systemctl mask plymouth-quit-wait.service

    再次重启后，使用system-analyze blame查询可以发现plymouth-quit-wait.service已经被禁用

    4 安装VMware tools

    通常情况下在VMware安装系统时会自动安装好VMware tools,此外还可以通过以下命令来进行安装

    vmware-toolbox-cmd --version    ##查看是否安装VMware tools

sudo apt install open-vm-tools-desktop
sudo apt install open-vm-tools

    博主在安装好vmware tools后尝试过多种方法仍无法解决文件的拖拽，但此功能有时候在安装好系统后就可直接使用。

    5 共享文件夹设置

    当需要在linux和windows平台下实现共享文件夹只需要在VMware界面依次选择虚拟机>设置>选项>共享文件夹进行如下设置

    需要注意在windows下需要将共享文件夹的属性设置成共享式

    设置好后可以在ubuntu系统/mnt/hgfs/路径下找到共享的文件夹。

    5.1 解决重启后共享文件夹消失问题

    在使用ubuntu22.04设置好共享文件夹后，重启后发现共享文件夹消失。实际上共享文件夹已经设置成功，只是没有自动挂载。使用下面的命令可以查看是否设置好共享文件夹，如果设置好则会输出共享文件夹名

    vmware-hgfsclient   #回车后输出目录名virtual_share

    设置开机自动挂载

    $ sudo vim /etc/rc.local

    输入以下内容

    #!/bin/sh -e
sudo sudo vmhgfs-fuse .host:/ /mnt/hgfs -o allow_other

    编辑好后保存退出，进入etc目录，给文件添加可执行权限

    chmod +x rc.local

    设置好后每次启动就可以自动挂载共享文件夹了

    6 解决虚拟机内删除文件，磁盘空间不减少问题

    当安装好VMware-Tools时可直接输入以下命令

    sudo /usr/bin/vmware-toolbox-cmd disk list
sudo /usr/bin/vmware-toolbox-cmd disk shrink /

    之后会弹出一个进度条，等待其结束即可
