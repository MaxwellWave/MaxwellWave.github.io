    0 前言

    log函数在无线通信中经常会用到，但是因为其涉及浮点数，而在Verilog中要实现浮点数就必须进行定点数量化，否则的话在硬件上无法实现。

    2 Verilog实现方法

    根据上一小结中可以知道对于任意的log函数都可以在log2的基础上通过换底公式实现，故而在Verilog中可以通过实现log2函数，并封装成一个单独的模块，在实现任意log函数的时候，只需要通过调用此模块即可。

    根据上述的公式推理，log2函数的实现可以拆分在整数部分和小数部分，整数部分即得到输入数据A中最高bit位的位置k（二进制格式数据），小数部分则可根据精度的要求，采用查表的方式进行实现。具体实现可拆分成如下步骤

    第一步：整数位计算，查找输入数据不为0的最高bit位k，即与博主之前写的查找输入数据首1的需求一致，故而实现方法可参考此文查找首1的三种方法 | MaxwellWave

    第二步：小数位计算，从k-1bit位开始，依次向后取m个bit数据，m的大小影响计算精度，m越大，精度越高。最终得到mbit的数据，将该数据作为查找表的索引即地址，查询查找表的数据即小数部分F。查找表中的具体值可以根据精度m使用matlab依次遍历打印，查找表的深度即位2^m，位宽根据定点小数位决定。

    从k-1位开始是因为在使用matlab打印小数数据的时候在内部将A的高位化简为了1，即小数值得计算公式为fra=log2(1+(0:2^m-1)/2^m)

    第三步：定点运算，如果结果定点为Fix16_8，即输入数据16bit，前8bit为整数位，后8bit为小数位，即查找表中的位宽深度为8bit。当得到整数k时，可将k左移8bit拓展成16bit的数据得到输出数据的整数部分，再与从ROM中读取的小数部分数据相加即可得到最终的log2函数的结果。

    定点数和浮点数的区别（计算机中的定点数与浮点数）

    • 定点数：小数点的位置在计算机的存储是约定好固定的，一个小数的整数部分和小数部分分别转换为二进制表示。如十进制的25.125，整数部分25用二进制表示为：11001；小数部分0.125使用二进制表示为.001；故而两者合起来为11001.001，而实际上计算机中不存储小数点，故而25.125的二进制为11001001，因为已经约定好小数位是低三位，所以这个8bit数据的前5bit就是整数，后3bit就是小数。如果小数位数的精度为3bit，那么小数位最多可以表示的范围为0~0.875。
    • 浮点数：计算机中使用浮点数表示小数类似于以前数学中用科学计数法表示较大的数。例如352.47=3.5247x10的2次方。178.125转化为二进制为 10110010.001，又可表示为：1.0110010001 乘以 2的111次方（111是7的二进制表示）。其中，

      • 10110010001 这部分被称作尾数(M)

      • 111这部分被称作阶码（P）

      • 正负被称作数符（S）: 0表示正数 ， 1表示负数。

    那么一个浮点数可以使用三部分表示：数符（S）,阶码（P）,尾数（M）。

    3 小数查找表的建立

    采用matlab进行定点数的计算，具体代码如下

    m = 5;
a = log2(1+(0:2^m-1)/2^m);

fid = fopen('a.txt','wt');
a_fix = fi(a,0,5,5)*2^m;
b = a_fix;
fprintf(fid,'data <= 5''d%d;\n',b);
fclose(fid);
    • m为小数部分的定点位宽
    • fi为matlab中的定点化函数，fi(data,s,w,f)。data为需要定点的浮点数，s为有符号和无符号数标志，w为数据位宽，f为定点小数的精度

    4 Verilog实现

    当输入数据位宽为32bit时，整数部分就可以用5bit表示，如果输出数据位宽限定为10bit，那么小数数据位宽即可定为5bit。故可参考Verilog实现对数运算log_verilog 对数运算-CSDN博客进行如下代码设计

    module log # (
parameter O_WIDTH_F = 5, //output fraction width
    parameter              PRECISION_W  = 5,
parameter I_WIDTH = 32, //input width
parameter O_WIDTH = O_WIDTH_F + $clog2(I_WIDTH) //output width
)(
 input                       clk,
 input [I_WIDTH-1 : 0]       i_data,
 output reg [O_WIDTH-1 : 0]  o_data
);

    
 wire  [I_WIDTH-1 : 0]       r_i_data;
    reg [$clog2(I_WIDTH)-1 : 0]  int_d, int_d_d0;
    reg [I_WIDTH-1     : 0]  fra_index;

    assign  r_i_data = i_data;

always @ (posedge clk) begin
 casez(r_i_data) 
        {{00{1'b0}},1'b1,{(I_WIDTH-01){1'b?}}}: begin int_d <= I_WIDTH - 01; fra_index <= r_i_data << 01; end
        {{01{1'b0}},1'b1,{(I_WIDTH-02){1'b?}}}: begin int_d <= I_WIDTH - 02; fra_index <= r_i_data << 02; end 
        {{02{1'b0}},1'b1,{(I_WIDTH-03){1'b?}}}: begin int_d <= I_WIDTH - 03; fra_index <= r_i_data << 03; end 
        {{03{1'b0}},1'b1,{(I_WIDTH-04){1'b?}}}: begin int_d <= I_WIDTH - 04; fra_index <= r_i_data << 04; end 
        {{04{1'b0}},1'b1,{(I_WIDTH-05){1'b?}}}: begin int_d <= I_WIDTH - 05; fra_index <= r_i_data << 05; end 
        {{05{1'b0}},1'b1,{(I_WIDTH-06){1'b?}}}: begin int_d <= I_WIDTH - 06; fra_index <= r_i_data << 06; end 
        {{06{1'b0}},1'b1,{(I_WIDTH-07){1'b?}}}: begin int_d <= I_WIDTH - 07; fra_index <= r_i_data << 07; end 
        {{07{1'b0}},1'b1,{(I_WIDTH-08){1'b?}}}: begin int_d <= I_WIDTH - 08; fra_index <= r_i_data << 08; end 
        {{08{1'b0}},1'b1,{(I_WIDTH-09){1'b?}}}: begin int_d <= I_WIDTH - 09; fra_index <= r_i_data << 09; end 
        {{09{1'b0}},1'b1,{(I_WIDTH-10){1'b?}}}: begin int_d <= I_WIDTH - 10; fra_index <= r_i_data << 10; end 
        {{10{1'b0}},1'b1,{(I_WIDTH-11){1'b?}}}: begin int_d <= I_WIDTH - 11; fra_index <= r_i_data << 11; end 
        {{11{1'b0}},1'b1,{(I_WIDTH-12){1'b?}}}: begin int_d <= I_WIDTH - 12; fra_index <= r_i_data << 12; end 
        {{12{1'b0}},1'b1,{(I_WIDTH-13){1'b?}}}: begin int_d <= I_WIDTH - 13; fra_index <= r_i_data << 13; end 
        {{13{1'b0}},1'b1,{(I_WIDTH-14){1'b?}}}: begin int_d <= I_WIDTH - 14; fra_index <= r_i_data << 14; end 
        {{14{1'b0}},1'b1,{(I_WIDTH-15){1'b?}}}: begin int_d <= I_WIDTH - 15; fra_index <= r_i_data << 15; end 
        {{15{1'b0}},1'b1,{(I_WIDTH-16){1'b?}}}: begin int_d <= I_WIDTH - 16; fra_index <= r_i_data << 16; end 
        {{16{1'b0}},1'b1,{(I_WIDTH-17){1'b?}}}: begin int_d <= I_WIDTH - 17; fra_index <= r_i_data << 17; end 
        {{17{1'b0}},1'b1,{(I_WIDTH-18){1'b?}}}: begin int_d <= I_WIDTH - 18; fra_index <= r_i_data << 18; end 
        {{18{1'b0}},1'b1,{(I_WIDTH-19){1'b?}}}: begin int_d <= I_WIDTH - 19; fra_index <= r_i_data << 19; end 
        {{19{1'b0}},1'b1,{(I_WIDTH-20){1'b?}}}: begin int_d <= I_WIDTH - 20; fra_index <= r_i_data << 20; end

        {{26{1'b0}},1'b1,{(I_WIDTH-27){1'b?}}}: begin int_d <= I_WIDTH - 27; fra_index <= r_i_data << 27; end 
        {{27{1'b0}},1'b1,{(I_WIDTH-28){1'b?}}}: begin int_d <= I_WIDTH - 28; fra_index <= r_i_data << 28; end 
        {{28{1'b0}},1'b1,{(I_WIDTH-29){1'b?}}}: begin int_d <= I_WIDTH - 29; fra_index <= r_i_data << 29; end 
        {{29{1'b0}},1'b1,{(I_WIDTH-30){1'b?}}}: begin int_d <= I_WIDTH - 30; fra_index <= r_i_data << 30; end 
        {{30{1'b0}},1'b1,{(I_WIDTH-31){1'b?}}}: begin int_d <= I_WIDTH - 31; fra_index <= r_i_data << 31; end 
        {{31{1'b0}},1'b1,{(I_WIDTH-32){1'b?}}}: begin int_d <= I_WIDTH - 32; fra_index <= {(I_WIDTH){1'b0}}; end 
        default:begin int_d <= {$clog2(I_WIDTH){1'b0}}; fra_index <= {PRECISION_W{1'b0}};end
 endcase
end

        wire [O_WIDTH_F -1 : 0] fra_data;        

fra_rom fra_rom_i
(
/*    input        */   .clk    (clk),
/*    input        */   .en     (1'b1),
/*    input   [4:0]*/   .addr   (fra_index[I_WIDTH-1:PRECISION_W]),
/*    output  [4:0]*/   .dout   (fra_data) 
);

always@(posedge clk) begin
    int_d_d0    <=  int_d;
end

always@(posedge clk) begin
    o_data  <=  {int_d_d0, fra_data};
end

endmodule

    疑问：在上述代码中，小数索引的截取是根据已知首1位置然后将输入数据左移相应位置后再截取对应精度的高位数据（即把精度数据移位到高位）作为索引进行查表。此处存在一个问题是，如果当首1的位置小于精度，那么此时将输入数据左移就会导致高位的数据可能是补0后的数据，而非实际上的值。如：如果输入数据为3，首1的位置应该是在1处，此时如果将数据左移31位，那么截取出来的数据应该是5’b10000，而不是实际的1，故而对实际的运算结果造成错误。在Verilog实现对数运算log_verilog 对数运算-CSDN博客采用了一种拼接的方法来避免此问题，但是无法通过VCS编译，因为位拼接符不支持负数。如果已知精度的话，把相应的精度case选项不进行移位即可。即使不采取此措施，也只是会对精度造成比较小的影响。
    答：上述理解是错误的，依然以输入数据为3为例，如果按照上述说法，小数位的值仅为1，如果不进行移位拓展送入到小数查找表中，那么查找到的值就为ROM中地址1的值。但是根据我们上述matlab打印的小数来看，地址1存储的实际上是log2(1+1/2^5)的定点化数，而实际上十进制3定点化的值应该是log2(1+1/2)，因为在matlab打印的时候为了能够遍历所有的数，故而除了2^5，故而对于输入数据3的话，定点数的计算公式应该为log2(1+1x2^4/2^5)，对应ROM中的地址为16。实际上也就是设计中的向左移位5bit，即小数位必须保证同比例的放缩。
