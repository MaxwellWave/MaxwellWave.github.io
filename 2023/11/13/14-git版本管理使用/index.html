    git版本管理使用

    1. Git下载与配置

    1.1. 在 Linux 上安装

    如果你想在Linux上用二进制安装程序来安装基本的 Git 工具，可以使用发行版包含的基础软件包管理工具来安装。以Fedora为例，如果你在使用它(或与之紧密相关的基于RPM的发行版，如RHEL或CentOS)，你可以使用dnf:

    $sudo dnf install git-all

    如果在基于Debin的发行版上，如Ubuntu，请使用apt

    $sudo apt install git-all

    安装完成后可在终端输入如下命令查询安装的版本

    $ git --version

    如果安装成功则会显示git版本号

    1.2 在 Windows 上安装

    Windows上安装可直接在官方网站上Git - Downloading Package (git-scm.com)直接下载，下载完成后在文件夹中右键即可看到 git bash

    1.3. Tortois Git 下载

    Tortois Git 是一个Git版本控制客户端，在Windows下带有GUI图形界面。安装过程中需要进行邮箱和用户名进行配置，该邮箱和用户名需要和github或gitee保持一致，安装完成后右键即可看到Tortois Git图标。

    但是安装完成后发现Tortois Git的图标不显示，这是应为在注册表中TortoiseGit图标覆盖排序不靠前，打开TortoiseGit的设置界面，找到图标覆盖→覆盖处理器→启动注册表编辑器

    可以看到该段文字中描述在注册表中还有其他的图标覆盖，总数限制是15个，所以超出15个之后的是不会生效的

    打开注册表之后，可以看到Tortoise相关的小图标，右键这些小图标重命名，在前面多加几个空格，让它们排序更靠前，空格越多排序越靠前，此处我还有OneDrive相关的小图标，但是我用不上，所以我可以直接删除，然后打开任务管理器，重启资源管理器，就可以正常显示小图标了

    1.4. Git 初次配置

    安装完成Git后需要进行初始化配置，每台计算机只需要配置一次，程序升级时会保留配置信息。第一个需要配置的是用户名和邮箱地址，这一点很重要，因为每一个Git提交都会使用这些信息，它们会写入到你的每一次提交中，不可更改

    $ git config --global user.name "John Doe"
$ git config --global user.email johndoe@163.com

    再次强调，如果使用了 --global 选项，那么该命令只需要运行一次，因为之后无论你在该系统上做任何事情， Git 都会使用那些信息。 当你想针对特定项目使用不同的用户名称与邮件地址时，可以在那个项目目录下运行没有 --global 选项的命令来配置。 很多 GUI 工具都会在第一次运行时帮助你配置这些信息。

    2. 获取Git仓库

    通常有两种获取Git项目仓库的方式

    1.将尚未进行版本控制的本地目录转换为Git仓库

    2.从其他服务器克隆一个已存在的Git仓库

    2.1 本地初始化仓库

    对于一个工程目录，如果希望使用Git来进行控制，那么可以进入该目录中输入以下命令进行初始化

    $ git init

    该命令将创建一个名为.git的子目录，这个子目录含有你初始化的Git仓库中所有的必须文件，这些文件是Git仓库的骨干。但是，在这个时候，我们仅仅是做了一个初始化的操作，项目里的文件还没有被跟踪。

    如果在一个已存在文件的文件夹（而非空文件夹）中进行版本控制，你应该开始追踪这些文件并进行初始提交。可以通过git add命令来指定所需的文件来进行追踪，然后执行git commit

    $ git add *.c
$ git add license
$ git commit -m "initial project version"

    2.2 远程仓库建立并获取

    通常项目代码除了放在本地，也会在远程服务器上进行部署，使得多个用户进行拉取和提交。常用的远程服务器为Github和Gitee，以下以Gitee上建立仓库进行举例。

    2.2.1 仓库建立

    当注册完Gitee账号后，点击右上角的+号，选择新建仓库

    进入新建仓库界面后，输入仓库名称后会自动填充仓库地址，默认地址与仓库名称一致，用户也可自行更改英文地址

    点击创建后Gitee会弹出一个初始教程界面，可参考该界面完成相应的操作

    此时远程仓库已经完成建立了，可以回到我们的本地目录下。

    2.2.2 获取远程仓库

    远程仓库的添加可以直接使用git clone命令进行拉取，如下

    $ git clone https://gitee.com/maxwell-wave/test.git

    输入以上命令后终端会弹出需要输入Gitee的用户名和密码的窗口，此时根据要求填写注册邮箱和密码即可完成拉取。为了避免每次提交和拉去可以采用ssh公钥的方式进行认证。具体方法如下

    2.2.3 ssh公钥设置

    打开终端，输入以下命令生成ssh key

    $ ssh-keygen -t -rsa -C "your email"

    按照提示，连续三次回车，即可生成ssh key。生成的ssh key会在~/.ssh/id_rsa_pub文件内，打开该文件复制文件内的公钥。

    进入gitee个人主页下的个人设置界面中，点击右下的安全设置一栏中的SSH公钥

    然后在点击确认，回到终端界面，输入以下命令判断是否田间成功

    $ ssh -T git@gitee.com

    一般来说会返回Hi XXX! You've successfully authenticated, but Gitee.com does not provide shell access.意思就是你已通过身份验证，但是gitee.com不提供shell访问。

    经过如上操作，如果发现还是会需要重复输入密码，那么可以进行如下操作，输入以下命令，让git记住账号和密码

    $ git config --global credential.helper store

    然后push或者pull下，会提示输入用户名和密码，输入第一次以后，就无需再次输入了。

    3.常用命令

    3.1 基础命令

    $ git help <command>				#获取git命令的帮助信息
$ git init 							#创建一个新的git仓库，其数据会存放在一个名为.git的目录下

$ git status						#显示当前仓库的状态
$ git status -s						#简要显示当前仓库状态

$ git add <filename>				#添加文件到暂存区
$ git commit						#提交，弹出提交说明，提前务必保证文件均已add过
$ git commit -m "info"				#将提交信息与命令放在同一行
$ git commit -a -m "info"			#跳过git add，将已跟踪的文件暂存起来一并提交

$ git log							#显示历史日志

$ git diff <filename>				#显示与暂存区文件的差异
$ git diff <revision> <filename>	#显示某个文件两个版本之间的差异
$ git checkout <revision>			#更新HEAD和目前的分支

    3.2 分支与合并

    $ git branch					#显示分支
$ git branch <name>				#创建分支
$ git checkout -b <name>		#创建分支并切换到该分支，相当于 git branch <name>; git checkout <name>
$ git merge <revision>			#合并到当前分支
$ git mergetool					#使用工具来处理合并冲突
$ git rebase					#将一系列补丁变基（rebase）为新的基线

    3.3 远端操作

    $ git remote												#列出远端
$ git remote add <name> <url>								#添加一个远端

$ git push <remote> <local branch>:<remote branch>			#对象传送至远端并更新远端引用
$ git push origin master

$ git branch --set-upstream-to=<remote>/<remote branch>		#创建本地和远端分支的关联关系

$ git fetch <remote name>  									#从远端获取对象/索引
$ git merge <remote name>/<remote branch>
$ git merge origin/master

$ git pull													#相当于 git fetch; git merge	
$ git clone													#从远端下载仓库

    3.4 打标签

    $ git tag									#列出标签

$ git tag <tag name >						#创建轻量标签，没有其他信息
$ git tag -a <tag name > -m "info" 			#创建附注标签
$ git show <tag name >						#显示标签信息与之对应的提交信息
    
$ git push origin <tag name >				#将标签提交到远程仓库
$ git push origin --tags 					#将所有标签提交到远程仓库

$ git tag -d <tag name>						#删除本地标签
$ git push origin --delete <tag name>		#删除远程仓库标签

    4 问题与结局

    4.1 描述

    在使用 hexo 将博客部署到 GitHub 仓库时出现如下错误

    git SSL certificate problem: unable to get local issuer certificate

    4.2 原因

    此问题是由于没有配置信任的服务器 HTTPS 验证。默认 cURL 被设为不信任任何 CAs ，就是说，它不信任任何服务器验证。

    4.3 解决

    输入以下命令即可解决

    git config --globle http.sslVerify         false
